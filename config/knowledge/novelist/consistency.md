# 连贯性保证机制

为确保长时间创作的故事连贯性：

## 写前必读

每次开始写新章节前：

**标准模式（≤50章）：**
1. 阅读 `00-大纲.md` 中所有已完成章节的摘要
2. 读取上一章文件，了解当前悬念
3. 检查人物状态（位置、情绪、关系）

**巨著模式（>50章）：**
按 `volume-arc-template.md` 中的"滑动窗口写作法"执行——只读总纲、当前卷纲、悬念追踪器活跃部分、上一章正文、前3-5章摘要。绝不试图读取全部历史章节。

## 穿针引线

在新章节中：
- 呼应前文埋下的伏笔和线索
- 提及之前发生的事件（自然融入）
- 让人物行为与之前保持一致

## 人物状态跟踪

注意人物在各章节中的变化和成长：
- 位置变化（人在哪里）
- 情绪状态（当前心情）
- 关系变化（与其他角色关系）
- 能力变化（获得新技能/信息）

## 悬念线延续

确保主线悬念逐步推进：
- 每章至少回应一个旧悬念
- 提出新悬念或升级现有悬念
- 不要遗忘任何未解的悬念

## 一致性检查清单

- [ ] 人物行为符合其性格设定
- [ ] 前后伏笔有呼应，逻辑闭环
- [ ] 高潮低谷分布合理，节奏恰当
- [ ] 时间线连贯（没有时间跳跃错误）
- [ ] 场景转换自然（没有凭空出现）

## 悬念一致性检查

在写每一章之前，必须检查悬念追踪器（`02-悬念追踪器.md`）：

### 写前检查
- [ ] 是否有 planted 超过20章未进入 fermenting 的悬念？如有，本章必须推进它
- [ ] 是否有章节钩子超过3章未回应？如有，本章必须回应
- [ ] 当前活跃的 fermenting 悬念数量是否充足（至少3条）？

### 写后检查
- [ ] 本章是否回应或推进了至少1条旧悬念？
- [ ] 本章是否埋设或强化了至少1条悬念？
- [ ] 本章揭示的悬念是否有充足的前期铺垫（至少2次发酵记录）？
- [ ] 悬念追踪器是否已更新？

### 卷间过渡检查（巨著模式）
- [ ] 本卷的所有卷级悬念是否已 revealed 或 closed？
- [ ] 未闭合的卷级悬念是否已提升为全书级悬念？
- [ ] 是否有相关的篇章级悬念因卷级悬念闭合而遗漏？
- [ ] 下一卷的开场是否回应了上一卷的结尾钩子？

## 跨卷连贯性保证（巨著模式）

详见 `volume-arc-template.md` 中的"滑动窗口写作法"和"卷间过渡"章节。

### 卷首重置要点
每卷第一章需要通过行动和对话自然地提醒读者：
- 上卷结局后主角的当前状态
- 当前的核心矛盾
- 避免信息倾倒式的"前情回顾"
